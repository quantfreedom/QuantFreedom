    Free Bot Finish¶

    Make sure you read the entire bullet point before you start doing what the bullet point says.

    Donnload WINSCP¶

    • Download winscp https://winscp.net/eng/downloads.php
    • Scroll down and click the download winscp green button and it should auto download, then launch the install
    • Click install for all users
    • Click on accept
    • Choose typical then next
    • Then commander then next
    • Then install
    • Uncheck open get started page and uncheck launch then hit finish

    Github token¶

    • In the upper right corner of github click on your profile icon and go to settings at the bottom of the menu
    • now go all the way down to the bottom and on the left side click dev settings
    • then on the left side click personal access tokens then click on token classics
    • now click on generate a new token on the upper right side next to revoke all and click on generate new token classic
    • then select the main repo that will select all the options under repo

    • then scroll to the bottom and choose create token
    • make sure you copy and save this somewhere because you will need it to run the strategy

    AWS Launch Template¶

    • Log into AWS and go to the console home page and click on EC2
    • On the left side click on security groups under network and security

    • Click on the blue name under secuirty group id which should have the name default under secuirty group name

    • Now we are going to edit the inbound rules in the inbound rules then click edit inbound rules

    • Type will be ssh ... source will be ipv4 and then hit save rule

    • Now click on the outbound tab click edit outbound rules
    • change the type to all traffic and destination to anywhere ipv4 then save rules

    • On the left side menu click on launch templates under instances and then click the orange create launch template in the upper right corner

    • Give your template a name under launch template name

    • In the next section choose ubuntu

    • For instance type choose the t2 free tier option

    • next in key pair login click on create new key pair

    • Then name the key pair something and select ppk and then click create key pair

    • Make sure you save the download somewhere other than your desktop because you will need to use it later on.
    • In network settings choose the security group that we made earlier

    Adding Github Token¶

    • Open this script
    • On line 7 change where is says %%%%%%%GITHUB_TOKEN with yoru github token
    • On line 13 replace %%%%%%%STRAT_FOLDER with the name of the folder of the strat you want to run ... Ex: 02_rsi_rising_falling ... you can get the name of the folder from the main repo https://github.com/QuantFreedom1022/live_bots
    • On line 14 replace %%%%%%%RUN_FILE.py with the main.py in the folder of the strat you want to run ... Ex: rsi_main.py it will always be a file with main in the name of it
    • Now replace the %%%%%%%API_KEY with your api key
    • Replace the %%%%%%%SECRET_KEY with your secret key
    • Now replace %%%%%%%USE_TEST_NET with a true if you want to use the testnet and a false if you want to use the main net ... all lower case
    • Finally replace %%%%%%%%EXCHANGE_NAME with the name of the exchange you want to trade on ... Ex: mufex ... all lower case

    • Go back to the amazon template website

    AES Advanced Details¶

    • Scroll down to the bottom and open advanced details

    • Scroll down to the part that says Metadata version and make sure you have v2 only selected

    • Then go down to the bottom and paste in the aws sript that you made
    • Click on the create template create launch template

    Running The Bot¶

    • now in the green section click on the link

    • then click on actions on the right side and click launch instance from template

    • then on the right side click on the orange button launch instance
    • then in the green success box click the link
    • then under instance id you click that blue link

    • copy the ipv4 dns

    WINSCP¶

    • Open winscp
    • in the login screen if you can't edit the host name click edit
    • in the host name field past in the public ipv4 dns
    • Paste it into the host name

    • change the user name to ubuntu
    • clcik on the button advanced and on the left side go to ssh authentication
    • now find the private key file that you downloaded and hit ok

    • now hit save and then hit login and hit yes

    Checking The Logs¶

    • You may have to wait 3 to 5 mins for the live bots folder to appear
    • Once the folder shows up you can open the folder and then inside of that folder there should be a logs folder

    • If you don't see a logs folder just wait another min or two and click the refresh button until you see the logs folder.
    • Click on the log file to see what the bot is doing. open logs and you should see that the bot is running
